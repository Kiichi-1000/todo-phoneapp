## TodoApp 要件定義書

### 1. 概要

- **システム名**: TodoApp（ワークスペースベースのタスク管理アプリ）
- **プラットフォーム**: Expo / React Native（iOS / Android / Web 想定）
- **データストア**: Supabase（PostgreSQL）
- **目的**: 日付ごとの「ワークスペース」にタスクを整理し、4分割グリッド/ポストイット/ノートといった視覚的な表現で、今日〜未来のタスク計画と進捗把握をしやすくする。

### 2. 対象ユーザー

- 日付ベースでタスクを整理したい個人ユーザー
- スマートフォン中心でタスクを管理したいユーザー
- 「どのエリアがどれくらい進んでいるか」を感覚的に把握したいユーザー

### 3. 用語定義

- **ワークスペース**: 1日を単位としたタスクボード。`date` と `type` を持ち、タイトル（例: 2026年3月17日 (火)）で表示される。
- **ワークスペースタイプ**:
  - `four_grid`: 4分割グリッドモード。`top_left / top_right / bottom_left / bottom_right` の4エリアにタスクを配置。
  - `individual`: ポストイット個別モード。ホワイトボード上に付箋を並べるイメージでタスクを管理。
  - `note`: ノートモード（コード上では「開発中」と明示）。
- **エリア名**: 4分割グリッドモードで各エリアに付与されるラベル（例:「重要・緊急」「今週中」など）。ユーザーが編集可能。
- **タスク（Todo）**: ワークスペースに紐づく1つの作業項目。完了フラグや完了日時を持つ。
- **ユーザー設定**: `default_workspace_type` など、ワークスペースの初期表示に関わる設定。

### 4. 機能要件

#### 4.1 ワークスペース管理

- **F-WS-01**: ユーザーは新規ワークスペースを作成できる。
  - 入力項目: タイトル、日付（YYYY-MM-DD）、ワークスペースタイプ（現状は `four_grid` のみUIから選択可能）。
  - 作成後は `/workspace/{id}` 画面へ遷移する。
- **F-WS-02**: 1日につき1ワークスペース（UNIQUE制約）として扱う。
  - 同じ `date` のワークスペースがすでに存在する場合、新規作成ではなく既存を取得・更新する挙動とする。
- **F-WS-03**: `default_workspace_type` に基づき、ホームタブ（`app/(tabs)/workspace.tsx`）では「今日」のワークスペースを自動的に表示する。
- **F-WS-04**: ユーザーはカレンダー/日付リストから任意の未来日を選択し、その日のワークスペースを新規作成または既存のものを開くことができる。
- **F-WS-05**: 設定画面でワークスペースタイプを変更した場合、未来日や新規作成されるワークスペースの `type` が変更される。

#### 4.2 ワークスペースビュー（4分割グリッドモード）

- **F-GRID-01**: 画面は 2×2 の4エリアに分割され、それぞれにエリア名・進捗率・タスク一覧・追加UIを表示する。
- **F-GRID-02**: 各エリア名はタップで編集でき、変更内容は Supabase の `workspaces.area_titles` に保存される。
- **F-GRID-03**: 各エリアに紐づくタスクの完了率（完了タスク数 / 総タスク数）が % で表示される。
- **F-GRID-04**: ユーザーは各エリアに対してタスクを追加できる。
  - 入力中のタスクはポストイット風のUIで表示される。
  - 空エリアでは「タップで追加」ヒント付きのエリア全体をタップすると追加入力が開く。
  - タスク追加後、該当ワークスペースのタスク一覧と進捗率が即時更新される。
- **F-GRID-05**: 各タスクはチェックボックスで完了／未完了を切り替えられる。
  - 完了時には `is_completed = true` と `completed_at` に日時を保存。
  - 未完了に戻した場合は `completed_at` を `null` にする。
- **F-GRID-06**: 各タスクは削除ボタンから削除できる。
  - 削除後、ローカル状態とSupabaseの両方から該当レコードを取り除く。
- **F-GRID-07**: 各タスクは長押し・矢印ボタン操作によって同一エリア内で並び順を変更できる。
  - 実装方法は `created_at` のスワップ or `order` カラムのスワップ（画面ごとに異なる方式を採用）。

#### 4.3 ポストイット個別モード（individual）

- **F-IND-01**: ワークスペースタイプが `individual` の場合、画面はホワイトボード風の1画面として表示される。
- **F-IND-02**: 右下の＋ボタンから新しいポストイットタスクを追加できる。
  - 追加時、`grid_area = null` とし、ランダムな座標を付与するロジックを持つ。
- **F-IND-03**: ポストイットタスクも完了／未完了フラグを持ち、チェックボックスで切り替え可能。
- **F-IND-04**: ポストイットも長押し＋矢印操作で並び替え可能（`created_at` ベース）。

#### 4.4 ノートモード（今後の拡張）

- **F-NOTE-01**: ワークスペースタイプが `note` の場合は「ノートモード（開発中）」というプレースホルダーを表示する。
- **F-NOTE-02**: 将来的にリッチテキストまたはシンプルなメモとして1日単位のノートを保存できることを想定する（詳細仕様は未定）。

#### 4.5 日付ナビゲーション／カレンダー

- **F-CAL-01**: ユーザーはスワイプジェスチャー（左右）で前日／翌日のワークスペースへ移動できる。
- **F-CAL-02**: 日付選択モーダルでカレンダーを表示し、未来の日付を選んで新規ToDo作成用のワークスペースを作成できる。
- **F-CAL-03**: ワークスペース一覧から「ToDoが存在する日付」をカウントし、「Xページ」としてヘッダーに表示する。

#### 4.6 タスク統計（Statisticsタブ）

- **F-STAT-01**: ユーザー設定の `default_workspace_type` に応じて、該当タイプのワークスペースに紐づくタスクのみを統計対象とする。
- **F-STAT-02**: 以下の統計情報を表示する。
  - 今日完了したタスク数
  - 直近1週間で完了したタスク数
  - 当月に完了したタスク数
  - 全期間の完了タスク数
  - 全期間の未完了タスク数
  - 完了率（完了数 / 全タスク数 × 100）
- **F-STAT-03**: プル・トゥ・リフレッシュで統計情報を再取得できる。

#### 4.7 ユーザー設定

- **F-SET-01**: 初回起動時に `user_settings` が存在しない場合、`default_workspace_type = four_grid` で自動作成する。
- **F-SET-02**: アプリがフォアグラウンドに戻った場合、`user_settings` を再読み込みし、ワークスペースの表示タイプを追従させる。
- **F-SET-03**: 設定画面（別タブ）でデフォルトワークスペースタイプを変更できる（詳細UIは別途仕様）。

### 5. 非機能要件

#### 5.1 パフォーマンス

- スマートフォン（ミドルレンジ端末）で快適に動作すること。
- スワイプアニメーションやモーダル表示は 200ms 前後のアニメーション時間で違和感のない挙動とする。

#### 5.2 ユーザビリティ

- タッチ操作に最適化されたUIとし、タップ領域・ボタンサイズは十分な大きさを確保する。
- 完了タスクには取り消し線や淡色表示を用いて完了状態が視覚的に分かるようにする。
- エリア名の編集・タスク編集・ポストイット追加など、編集操作はモバイルでストレスなく行えるシンプルなフローとする。

#### 5.3 信頼性

- Supabaseへの接続が失敗した場合、ユーザーにエラー内容と対処を分かりやすく表示する。
- エラー時にはローカル状態を保護しつつ、再読み込み処理（`loadWorkspace()` など）で復旧を試みる。

### 6. 画面一覧（高レベル）

- **ホームタブ / 日付ワークスペース画面**
  - 今日〜未来の日付のワークスペース表示（デフォルトは今日）。
  - スワイプで日付移動、カレンダーモーダルで日付選択。
  - ワークスペースタイプごとのUI出し分け。
- **ワークスペース詳細画面 `/workspace/{id}`**
  - 4分割グリッドで特定ワークスペースのタスクを集中管理。
  - タイトルバーに戻るボタンとワークスペース名を表示。
- **ワークスペース作成画面 `/workspace/create`**
  - タイトル・日付・タイプ選択フォーム。
- **統計画面（Statisticsタブ）**
  - タスク完了数・完了率のサマリ表示。
- **設定画面（Settingsタブ）**
  - デフォルトワークスペースタイプその他の設定（詳細は別仕様）。

### 7. データモデル（簡易）

- **workspaces**
  - `id`: PK
  - `title`: 文字列
  - `date`: 日付（1日1ワークスペースのユニーク制約）
  - `type`: `four_grid | individual | note`
  - `area_titles`: JSON（`top_left / top_right / bottom_left / bottom_right`）
  - `created_at`, `updated_at`
- **todos**
  - `id`: PK
  - `workspace_id`: FK → workspaces.id
  - `content`: タスク内容
  - `grid_area`: `top_left | top_right | bottom_left | bottom_right | null`
  - `order` または `created_at` による並び順
  - `is_completed`: boolean
  - `completed_at`: timestamp | null
  - `position_x`, `position_y`: ポストイット個別モード向け座標（null許容）
  - `due_date`: 期日（将来的な利用を想定）
- **user_settings**
  - `id`: PK
  - `default_workspace_type`: `four_grid | individual | note`
  - `created_at`, `updated_at`

### 8. 今後の拡張ポイント（メモ）

- 期日（`due_date`）に基づくリマインドやソート。
- ノートモードの実装（リッチテキスト / マークダウンなど）。
- 4分割エリアのテンプレートプリセット（例: Eisenhowerマトリクス）。
- 複数デバイス間同期のUX改善（オフライン対応）。
